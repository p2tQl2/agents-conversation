# 安全政策

## 报告安全漏洞

如果你发现了安全漏洞，请不要在公开的 GitHub Issues 中报告。相反，请通过以下方式私下报告：

1. **发送电子邮件**到项目维护者（邮箱地址待补充）
2. **标题**：`[SECURITY] 漏洞报告`
3. **包含以下信息**：
   - 漏洞的详细描述
   - 受影响的版本
   - 复现步骤（如果可能）
   - 潜在的影响
   - 建议的修复方案（如果有）

## 安全最佳实践

### 对于用户

- 定期更新到最新版本
- 不要在配置文件中存储敏感信息（如密钥、令牌）
- 使用环境变量存储敏感数据
- 限制 HTTP 服务器的绑定地址（默认为 `127.0.0.1`）
- 在生产环境中使用防火墙规则

### 对于开发者

- 不要在代码中硬编码敏感信息
- 使用 `.gitignore` 排除敏感文件
- 定期审查依赖项的安全更新
- 遵循安全编码实践
- 在提交前进行代码审查

## 依赖项安全

我们使用以下工具来监控依赖项的安全性：

- `npm audit` - 检查已知漏洞
- GitHub 的依赖项安全警报

定期运行以下命令检查依赖项：

```bash
npm audit
npm audit fix
```

## 响应时间

我们承诺在收到安全报告后的不知道多少小时内做出回应。

## 披露政策

- 我们将与报告者合作以了解和解决问题
- 在发布修复之前，我们会给予合理的时间进行修复
- 一旦修复发布，我们将公开确认该漏洞

感谢你帮助我们保持项目的安全。
